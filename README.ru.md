soundcheck - десктопный музыкальный плеер на WPF и .NET 10. Играет локальную
библиотеку, хранит всё в одном файле SQLite и поставляется одним самодостаточным
`.exe`. Интерфейс на русском и английском, переключается в любой момент.

## Возможности

- Кроссфейд между треками, перемешивание/повтор и очередь, которая переживает перезапуск
- 10-полосный эквалайзер с пресетами и своим слотом
- Плейлисты: создать / переименовать / удалить, обложка, порядок перетаскиванием, объединение, импорт/экспорт `.m3u`
- Редактор тегов (название / исполнитель / альбом + обложка), из библиотеки или файлового менеджера
- Файловый менеджер с пакетным переименованием по тегам
- Поиск, сортировка и перетаскивание файлов из проводника
- Полноэкранный режим «сейчас играет» с живым спектром
- Статистика: время прослушивания, включения, топ треков, история
- Акцентный цвет из обложки, размытый фон, иконка в трее, таймер сна, запуск с Windows

## Горячие клавиши

| Клавиша | Действие |
| --- | --- |
| `Space` | Пауза / воспроизведение |
| `←` / `→` | Предыдущий / следующий трек |
| `Enter` | Играть выбранный трек |
| `Esc` | Закрыть текущую панель |

## Сборка и запуск

Нужен Windows и .NET 10 SDK (только для сборки - опубликованная версия самодостаточна).

```
dotnet run -c Debug
```

Чтобы получить один самодостаточный `.exe`:

```
dotnet publish -c Release
```

Файл появится в `bin/Release/net10.0-windows/win-x64/publish/soundcheck.exe`. Копируй
куда угодно и запускай - .NET на машине не нужен.

## Данные

Библиотека, настройки, плейлисты, статистика и история - в одном файле:

```
%AppData%\SoundCheck\library.db
```

Резервная копия и восстановление - в Настройки → Данные. Аудиофайлы не трогаются,
пока ты сам не отредактируешь их теги.

## На чём сделано

WPF на .NET 10, [NAudio](https://github.com/naudio/NAudio) (звук, эквалайзер, FFT),
[TagLibSharp](https://github.com/mono/taglib-sharp) (теги) и
[Microsoft.Data.Sqlite](https://learn.microsoft.com/dotnet/standard/data/sqlite/) (библиотека).
